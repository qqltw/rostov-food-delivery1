# Деплой на Amvera Cloud

Проект уже подготовлен к деплою: см. [`amvera.yml`](amvera.yml).

- **Сборка:** `npm run build` (Vite собирает фронтенд в `dist/`)
- **Запуск:** `npm run db:deploy && NODE_ENV=production npm run start`
  (применяет миграции Prisma, затем поднимает Express-сервер на порту `3000`)
- **Порт контейнера:** `3000`

> `NODE_ENV=production` обязателен — иначе сервер поднимет dev-режим Vite вместо
> раздачи собранного `dist` (см. [`server.ts`](server.ts) `startServer()`).

---

## 1. Создать PostgreSQL в Amvera

1. В панели Amvera → **Создать проект** → тип **PostgreSQL**.
2. При создании задайте:
   - имя проекта БД (например `foodb`) — оно попадёт в хост,
   - имя пользователя (например `amvera`; имя `postgres` зарезервировано),
   - пароль,
   - имя базы данных.
3. После создания откройте вкладку **«Информация»** у проекта БД — там будет
   **внутреннее доменное имя** вида:

   ```
   amvera-<ваш-логин>-cnpg-<имя-проекта-бд>-rw
   ```

   - `-rw` — чтение и запись (нужен нам),
   - `-ro` — только чтение (горячий резерв),
   - `-r`  — чтение с любой ноды.

## 2. Собрать строку подключения (DATABASE_URL)

Формат (внутри сети Amvera SSL не нужен → `sslmode=disable`):

```
postgresql://<user>:<password>@amvera-<логин>-cnpg-<имя-бд>-rw:5432/<dbname>?schema=public&sslmode=disable
```

Пример:

```
postgresql://amvera:MyPass123@amvera-myuser-cnpg-foodb-rw:5432/foodb?schema=public&sslmode=disable
```

## 3. Создать проект приложения и задать переменные

1. В панели Amvera → **Создать проект** → окружение **Node**, npm v20
   (этого же требует [`amvera.yml`](amvera.yml)).
2. Залейте код: `git push amvera master` **или** загрузите через панель.
3. Во вкладке **«Переменные и секреты»** приложения добавьте:

   | Переменная       | Значение                                  | Тип     |
   |------------------|-------------------------------------------|---------|
   | `DATABASE_URL`   | строка подключения из шага 2              | секрет  |
   | `GEMINI_API_KEY` | ключ Gemini (если используете AI)         | секрет  |
   | прочие из [`.env.example`](.env.example) | по необходимости           | секрет  |

   > `NODE_ENV=production` уже выставляется командой запуска в `amvera.yml`,
   > отдельно задавать не обязательно.

4. Запустите сборку/деплой. При старте автоматически выполнится
   `prisma migrate deploy` — таблицы создадутся из [`prisma/migrations`](prisma/migrations).

## 4. Наполнить базу данными (seed)

После первого успешного деплоя БД пустая. Заполнить её можно:
- кнопкой **SEED DATA** в самом приложении (см. README), либо
- локально, указав боевой `DATABASE_URL`, командой: `npx tsx prisma/seed.ts`.

---

## Чек-лист перед деплоем

- [ ] Создан PostgreSQL-проект, скопировано внутреннее имя `...-rw`
- [ ] `DATABASE_URL` добавлен в секреты приложения
- [ ] Код залит, сборка прошла (`npm run build`)
- [ ] В логах деплоя видно успешный `prisma migrate deploy`
- [ ] Приложение открывается, БД наполнена через SEED
